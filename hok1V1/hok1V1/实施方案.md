## 1.7 归一化

所有连续数值进入网络前必须归一化。

必须归一化：

```text
hp / max_hp
ep / max_ep
money
exp
level
location_x
location_z
distance
attack_range
sight_area
move_speed
attack_speed
skill_cooldown
skill_cooldown_max
tower_hp
crystal_hp
frame_no
```

推荐工具类：

```python
class FeatureNormalizer:
    def clip01(self, x):
        return max(0.0, min(1.0, x))

    def ratio(self, value, max_value):
        if max_value <= 0:
            return 0.0
        return self.clip01(value / max_value)

    def min_max(self, value, min_value, max_value):
        if max_value <= min_value:
            return 0.0
        return self.clip01((value - min_value) / (max_value - min_value))
```

坐标归一化：

```python
location_x_norm = min_max(location_x, -60000, 60000)
location_z_norm = min_max(location_z, -60000, 60000)
```

# 2. 奖励设计

## 2.1 总原则

第一版 reward 保持简单，不做复杂规则堆叠。

目标：

```text
能让 PPO 学到：
1. 保持血量优势；
2. 清兵发育；
3. 推塔；
4. 减少死亡；
5. 通过击杀或压制创造推塔机会。
```

不在第一版加入：

```text
技能命中奖励
技能空放惩罚
走位奖励
控线奖励
追击奖励
撤退奖励
复杂塔下博弈奖励
复杂站位奖励
```

这些只做监控，后续 ablation 再加入。

---

## 2.2 Stage A：基础 reward

第一版只实现 Stage A。

奖励项：

```text
hp_point
tower_hp_point
money / gold
exp
last_hit
death
kill
forward
win_or_lose
```

权重：

```python
REWARD_WEIGHTS_STAGE_A = {
    "hp_point": 2.0,
    "tower_hp_point": 10.0,
    "money": 0.008,
    "exp": 0.008,
    "last_hit": 0.5,
    "death": -1.0,
    "kill": 0.5,
    "forward": 0.03,
    "win_or_lose": 3.0,
}
```

`forward` 必须小：

```python
forward <= 0.05
```

---

## 2.3 差分奖励

### 血量奖励

```python
hp_reward = (
    prev.enemy_hp_ratio - cur.enemy_hp_ratio
) - (
    prev.self_hp_ratio - cur.self_hp_ratio
)
```

含义：

```text
敌方掉血 → 正奖励
自己掉血 → 负奖励
```

---

### 防御塔奖励

```python
tower_reward = (
    prev.enemy_tower_hp_ratio - cur.enemy_tower_hp_ratio
) - (
    prev.self_tower_hp_ratio - cur.self_tower_hp_ratio
)
```

含义：

```text
敌方塔掉血 → 正奖励
己方塔掉血 → 负奖励
```

---

### 经济奖励

```python
gold_reward = (
    cur.self_money - prev.self_money
) - (
    cur.enemy_money - prev.enemy_money
)
```

---

### 经验奖励

```python
exp_reward = (
    cur.self_exp - prev.self_exp
) - (
    cur.enemy_exp - prev.enemy_exp
)
```

---

## 2.4 稀疏奖励

### 击杀奖励

```python
kill_reward = cur.self_kill_cnt - prev.self_kill_cnt
```

### 死亡惩罚

```python
death_reward = cur.self_dead_cnt - prev.self_dead_cnt
```

最终加权时：

```python
reward += death_weight * death_reward
```

其中：

```python
death_weight = -1.0
```

### 补兵奖励

```python
last_hit_reward = cur.self_last_hit_cnt - prev.self_last_hit_cnt
```

如果没有可靠的 last_hit 字段，先使用金币/经验变化替代，不要乱猜。

---

## 2.5 终局奖励

```python
if win == 1:
    win_reward = 1.0
elif win == -1:
    win_reward = -1.0
else:
    win_reward = 0.0
```

加权：

```python
reward += 3.0 * win_reward
```

终局奖励不宜过大，避免稀疏奖励主导训练。

## 2.9 RewardManager 结构

```python
class GameRewardManager:
    def reset(self, initial_obs):
        self.prev_stats = extract_stats(initial_obs)

    def step(self, obs, next_obs, action=None):
        cur_stats = extract_stats(next_obs)
        reward_items = compute_reward_items(
            prev=self.prev_stats,
            cur=cur_stats,
            action=action,
        )
        reward = weighted_sum(reward_items)
        self.prev_stats = cur_stats
        return reward, reward_items
```

`reward_items` 必须包含：

```text

reward_gold
reward_exp
reward_last_hit
reward_kill
reward_death
reward_forward
reward_win
reward_tower_danger
reward_push_assist
```

即使某项关闭，也返回 0，方便日志记录。

## 3.3 masked_max_pool

```python
def masked_max_pool(x, mask):
    # x: [B, K, D]
    # mask: [B, K]
    mask = mask.unsqueeze(-1)  # [B, K, 1]
    x = x.masked_fill(mask <= 0, -1e9)
    pooled = x.max(dim=1).values
    pooled = torch.where(torch.isfinite(pooled), pooled, torch.zeros_like(pooled))
    return pooled
```

## 3.7 action mask

### legal_action

```python
LABEL_SIZE_LIST = [12, 16, 16, 16, 16, 9]
```

拆分：

```python
button_mask, move_x_mask, move_z_mask, skill_x_mask, skill_z_mask, target_mask = split_mask(
    legal_action,
    LABEL_SIZE_LIST,
)
```

应用 mask：

```python
def apply_mask(logits, mask):
    return logits.masked_fill(mask <= 0, -1e9)
```

### sub_action_mask

采样流程：

```text
1. button_logits 应用 button_mask
2. 采样 / argmax 得到 button
3. 根据 button 读取 sub_action_mask[button]
4. 对 move_x / move_z / skill_x / skill_z / target 做二次 mask
5. 输出完整 action
```

## 3.8 PPO logprob

只计算有效动作头。

```text
button = Move:
    logprob = button + move_x + move_z

button = Skill:
    logprob = button + skill_x + skill_z + target

button = NormalAttack:
    logprob = button + target

button = Recall / None:
    logprob = button
```

不要把无意义动作头的 logprob 加入 PPO loss。

### predict
