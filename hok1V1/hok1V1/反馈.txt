技能弱的问题，先查这三项

技能能力弱不一定是网络表达能力差，可能是动作头和 mask 学错了。官方动作空间是层次化的：先选 button，再选方向 how，再选目标 who；而 sub_action_mask 会根据 button 过滤 move、skill、target。

重点检查：

1. PPO loss 是否只计算有效动作头？

例如普通攻击只算：

button + target

移动只算：

button + move_x + move_z

技能只算：

button + skill_x + skill_z + target

不要把所有 head 的 logprob 都加进 PPO loss，否则 skill_x、skill_z、move_x、move_z 会被大量无效动作污染。

2. exploit 是否正确使用 legal_action + sub_action_mask？

评估时不能只对每个 head 独立 argmax，要先确定 button，再用对应 button 的 mask 约束方向和目标。

3. 召唤师技能不要随机选

你当前代码里 init_config 是从所有召唤师技能里 random.choice，这会导致模型有时根本不是闪现，也不是终结。
如果你现在要提升“残血收割”，建议先固定：

summoner_skill_id = 80108  # 终结

如果你专门训练“闪现追杀”，再固定：

summoner_skill_id = 80115  # 闪现

不要一边随机召唤师技能，一边期待模型稳定学会闪现追击。
